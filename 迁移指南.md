# 数据迁移指南（方案 A：同一 docker-compose）

> 将旧 PostgreSQL 数据库中的 6 张核心表数据迁移到本项目的 Docker PostgreSQL。

---

## 前提条件

| 项目 | 要求 |
|------|------|
| 旧数据库 | PostgreSQL 14+，可通过端口访问（5432） |
| Docker | 已安装并运行 |
| Docker Compose | 已安装（v1.29+ 或 v2.x） |
| 本项目 | 已克隆到服务器（`git clone` 或 `scp` 上传） |

---

## 需要迁移的表

以下 6 张表是 `pipeline` 的核心数据表：

| # | 表名 | 说明 | 主键 |
|---|------|------|------|
| 1 | `books` | 书籍库 | `book_id` |
| 2 | `book_processing_states` | 断点续跑状态 | `(book_id, project_flag)` |
| 3 | `youtube_credentials` | YouTube OAuth 凭证 | `channel_name` |
| 4 | `modelscope_tokens` | AI 生图 Token | `channel_name` |
| 5 | `channel_runtime_settings` | 频道级运行配置 | `(channel_name, setting_key)` |
| 6 | `task_queue` | 任务队列 | `book_id` |

项目新增的 5 张表（`channels`、`channel_configs`、`run_tasks`、`run_task_logs`、`global_settings`）由 `init-db.sql` 自动创建，无需迁移。

## 方式一：使用迁移脚本（推荐）

### 第 1 步：配置 .env

> 如果通过 GitHub 部署，先克隆仓库：
> ```bash
> git clone https://github.com/YOUR_USER/audiobook-manager.git /opt/audiobook
> cd /opt/audiobook
> ```

```bash
cd /path/to/全部有声书yt频道统一管理网页程序
cp .env.example .env

# 编辑 .env
nano .env
```

`.env` 文件内容：

```env
# 数据库密码（必须修改）
POSTGRES_PASSWORD=你的强密码

# Web 密钥（必须修改，建议随机生成）
SECRET_KEY=$(openssl rand -hex 32)

# Web 访问地址
BASE_URL=http://你的服务器IP:8080
```

### 第 2 步：运行迁移脚本

```bash
# 赋予执行权限
chmod +x scripts/migrate_data.sh

# 方式 A：旧库无密码（trust 认证）
./scripts/migrate_data.sh

# 方式 B：旧库有密码
OLD_PG_PASSWORD=你的旧库密码 ./scripts/migrate_data.sh

# 方式 C：指定旧库连接参数
OLD_PG_HOST=127.0.0.1 \
OLD_PG_PORT=5432 \
OLD_PG_USER=postgres \
OLD_PG_DB=audiobook \
OLD_PG_PASSWORD=你的旧库密码 \
./scripts/migrate_data.sh
```

脚本会自动完成以下操作：
1. ✅ 测试旧库连接
2. ✅ 备份 6 张表的数据（仅数据，不含表结构）
3. ✅ 启动 Docker PostgreSQL 容器（自动执行 `init-db.sql`）
4. ✅ 验证 11 张表全部创建
5. ✅ 导入旧数据
6. ✅ 对比新旧库数据量

### 第 3 步：启动全部服务

```bash
docker-compose up -d
```

## 方式二：手动操作

如果不使用脚本，可按以下步骤手动迁移。

### 第 1 步：配置 .env

```bash
cd /path/to/全部有声书yt频道统一管理网页程序
cp .env.example .env
nano .env
# 设置 POSTGRES_PASSWORD 和 SECRET_KEY
```

### 第 2 步：备份旧库数据

```bash
# 设置旧库密码（如果有）
export PGPASSWORD="你的旧库密码"

# 仅导出 6 张核心表的数据（不含表结构）
pg_dump -h 127.0.0.1 -p 5432 -U postgres -d audiobook \
  --data-only --column-inserts --no-owner --no-privileges \
  --table=public.books \
  --table=public.book_processing_states \
  --table=public.youtube_credentials \
  --table=public.modelscope_tokens \
  --table=public.channel_runtime_settings \
  --table=public.task_queue \
  > /tmp/old_data.sql

# 确认备份文件不为空
ls -lh /tmp/old_data.sql
head -20 /tmp/old_data.sql
```

### 第 3 步：启动 Docker PostgreSQL

```bash
# 仅启动数据库服务
docker-compose up -d postgres

# 等待初始化完成（init-db.sql 会自动执行）
sleep 10

# 查看日志确认初始化成功
docker logs audiobook_postgres

# 验证表是否创建
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "\dt"
```

应看到 11 张表：
- 6 张核心表（books, book_processing_states, youtube_credentials, modelscope_tokens, channel_runtime_settings, task_queue）
- 5 张新增表（channels, channel_configs, run_tasks, run_task_logs, global_settings）

### 第 4 步：导入旧数据

```bash
# 导入数据（忽略错误以跳过可能的冲突）
docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook \
  -v ON_ERROR_STOP=off \
  < /tmp/old_data.sql
```

### 第 5 步：验证数据

```bash
# 检查各表数据量
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "
  SELECT 'books' as tbl, count(*) FROM books
  UNION ALL SELECT 'book_processing_states', count(*) FROM book_processing_states
  UNION ALL SELECT 'youtube_credentials', count(*) FROM youtube_credentials
  UNION ALL SELECT 'modelscope_tokens', count(*) FROM modelscope_tokens
  UNION ALL SELECT 'channel_runtime_settings', count(*) FROM channel_runtime_settings
  UNION ALL SELECT 'task_queue', count(*) FROM task_queue;
"

# 对比旧库数据量
psql -h 127.0.0.1 -U postgres -d audiobook -c "
  SELECT 'books' as tbl, count(*) FROM books
  UNION ALL SELECT 'book_processing_states', count(*) FROM book_processing_states
  UNION ALL SELECT 'youtube_credentials', count(*) FROM youtube_credentials
  UNION ALL SELECT 'modelscope_tokens', count(*) FROM modelscope_tokens
  UNION ALL SELECT 'channel_runtime_settings', count(*) FROM channel_runtime_settings
  UNION ALL SELECT 'task_queue', count(*) FROM task_queue;
"
```

### 第 6 步：启动全部服务

```bash
docker-compose up -d

# 查看所有服务状态
docker-compose ps

# 查看日志
docker-compose logs -f web
```

访问 `http://你的服务器IP:8080` 进入管理界面。

## 启动后配置

迁移完成后，还需要在 Web 界面或数据库中完成以下配置：

### 1. 同步全局设置

旧库中的 `channel_runtime_settings` 表里的 `__shared__` 配置（如 `HF_DATASET_ZIP_URLS`、`BUCKET_IDS`、`MODELSCOPE_TOKEN`）会随数据迁移一起导入。

同时，`init-db.sql` 在 `global_settings` 表中初始化了以下配置项（值为空，需要填写）：

| 配置项 | 说明 | 在 Web 界面填写 |
|--------|------|----------------|
| `HF_TOKEN` | Hugging Face API Token | 设置 → 全局设置 |
| `HF_DATASET_ZIP_URLS` | 音乐库下载链接 | 设置 → 全局设置 |
| `BUCKET_IDS` | HF Bucket ID 列表 | 设置 → 全局设置 |
| `SENSENOVA_API_KEY` | Sensenova/DeepSeek API 密钥 | 设置 → 全局设置 |
| `MODELSCOPE_TOKEN` | ModelScope API Token | 设置 → 全局设置 |

### 2. 注册频道

迁移后的 `youtube_credentials` 表已有 OAuth 凭证数据，但 `channels` 表是空的（新增表）。

需要在 Web 界面手动注册频道，或执行 SQL 关联：

```sql
-- 查看已有凭证的频道
SELECT channel_name, created_at FROM youtube_credentials;

-- 在 channels 表中注册这些频道
-- 方法一：通过 Web 界面「频道管理」→ 添加频道
-- 方法二：直接 SQL 插入（channel_name 需与 youtube_credentials 一致）
INSERT INTO channels (channel_name, display_name, oauth_status)
SELECT channel_name, channel_name, 'authorized'
FROM youtube_credentials
WHERE channel_name NOT IN (SELECT channel_name FROM channels)
ON CONFLICT (channel_name) DO NOTHING;
```

### 3. 验证 OAuth 状态

```sql
-- 检查频道和凭证的关联
SELECT
    c.channel_name,
    c.oauth_status,
    CASE WHEN yc.channel_name IS NOT NULL THEN '有凭证' ELSE '无凭证' END as credential_status,
    c.last_auth_at
FROM channels c
LEFT JOIN youtube_credentials yc ON c.channel_name = yc.channel_name;
```

## 常见问题

### Q: 导入时报 `duplicate key value violates unique constraint`

**原因**：新库已有同名数据。

**解决**：先清空目标表再导入：

```bash
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "
TRUNCATE TABLE books, book_processing_states, youtube_credentials,
             modelscope_tokens, channel_runtime_settings, task_queue CASCADE;
"
```

然后重新导入。

### Q: 导入时报 `column "xxx" does not exist`

**原因**：旧表结构和 `init-db.sql` 定义的不完全一致。

**解决**：

```bash
# 1. 查看旧表结构
psql -h 127.0.0.1 -U postgres -d audiobook -c "\d books"

# 2. 查看新表结构
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "\d books"

# 3. 如果旧表少了某些列，用 --column-inserts 导出会自动跳过不存在的列
# 如果旧表多了某些列，需要在新表中添加对应列：
# ALTER TABLE public.books ADD COLUMN IF NOT EXISTS extra_col text;
```

### Q: 旧库的数据库名或用户名不同

**解决**：通过环境变量指定：

```bash
OLD_PG_USER=myuser OLD_PG_DB=mydatabase ./scripts/migrate_data.sh
```

### Q: 迁移后 Web 界面看不到数据

**解决**：

1. 检查 `channels` 表是否有数据（Web 界面基于 `channels` 表展示）
2. 执行上面的「注册频道」SQL
3. 刷新 Web 页面

### Q: Docker 容器重建后数据会丢吗

**不会。** `docker-compose down` 只停止容器，数据保存在 Docker volume `postgres_data` 中。

只有 `docker-compose down -v`（带 `-v` 参数）才会删除 volume。

### Q: 如何备份新库

```bash
# 备份
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook \
  > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook \
  < backup_20250101.sql
```

### Q: 如何回滚到旧库

1. 停止 Docker 服务：`docker-compose down`
2. 启动旧库：`sudo systemctl start postgresql`
3. 修改 `.env` 指向旧库：
   ```env
   DATABASE_URL=postgresql://postgres:密码@127.0.0.1:5432/audiobook
   ```
4. 在 `docker-compose.yml` 中注释掉 `postgres` 服务和各服务的 `depends_on: postgres`
