## 1. 架构概览

### 1.1 服务拓扑

```
┌─────────────────────────────────────────────────────┐
│                   Docker Compose                     │
│                                                      │
│  ┌──────────────┐     ┌──────────────────────┐      │
│  │  audiobook   │     │   audiobook_postgres  │      │
│  │  _web        │────▶│   (postgres:16-alpine)│      │
│  │              │     │                      │      │
│  │ FastAPI:59386│     │  DB: audiobook       │      │
│  │ Pipeline线程 │     │  User: audiobook_app  │      │
│  │ DeepFilter   │     │  Port: 5432          │      │
│  └──────┬───────┘     └──────────────────────┘      │
│         │                                             │
│    ┌────┴────┐                                       │
│    │ Volumes │  output_data (Docker volume)          │
│    │         │  postgres_data (Docker volume)        │
│    │         │  ./data/music (bind mount)            │
│    │         │  ./data/deepfilter (bind mount)       │
│    │         │  ./pipeline (read-only mount)         │
│    └─────────┘                                       │
└─────────────────────────────────────────────────────┘
         │
         ▼
   外部网络访问 :59386
```

### 1.2 核心组件

| 组件 | 说明 |
|------|------|
| **web 容器** (`audiobook_web`) | FastAPI 应用 + 后台 Pipeline 线程，端口 59386，无需 Celery/Redis |
| **postgres 容器** (`audiobook_postgres`) | PostgreSQL 16 Alpine，仅自建数据库模式启动 |
| **Dockerfile.web** | 多阶段构建，Python 3.11-slim + ffmpeg + DeepFilter 自动下载 |
| **entrypoint.sh** | 容器启动入口，负责 DeepFilter 二进制和 BGM 音乐库的持久化初始化 |
| **init-db.sql** | 数据库首次启动自动执行，创建 13 张表 + 索引 + 初始全局设置 |

### 1.3 数据库表结构（13 张表）

init-db.sql 在 PostgreSQL 首次启动时自动执行，创建以下表：

| # | 表名 | 用途 |
|---|------|------|
| 1 | `books` | 书籍库（含 `book_status` 章节完成标记） |
| 2 | `audiobook_chapters` | 章节级 TG 缓存（含 worker_id/claimed_at/error_message） |
| 3 | `book_processing_states` | 断点续跑状态 |
| 4 | `youtube_credentials` | YouTube OAuth 凭证 |
| 5 | `modelscope_tokens` | AI 生图 Token |
| 6 | `channel_runtime_settings` | 频道级运行配置 |
| 7 | `task_queue` | 任务队列 |
| 8 | `channels` | 频道注册表 |
| 9 | `channel_configs` | 频道完整配置快照 |
| 10 | `run_tasks` | 运行任务记录 |
| 11 | `run_task_logs` | 任务日志 |
| 12 | `global_settings` | 全局共享设置 |
| 13 | `oauth_states` | OAuth 授权状态临时存储 |

## 2. Docker Compose 文件体系

本项目采用**基础文件 + 覆盖文件**的多文件组合模式，通过 `-f` 参数叠加。

### 2.1 文件清单

```
docker-compose.yml              ← 基础文件（仅 web 服务）
docker-compose.self-db.yml      ← 自建数据库覆盖（添加 postgres 服务）
docker-compose.external-db.yml  ← 外部数据库覆盖（无 postgres 服务）
docker-compose.lowmem.yml       ← 低配 VPS 资源限制覆盖
docker-compose.dev.yml          ← 开发环境覆盖（热重载 + 源码挂载）
```

### 2.2 组合方式

```
自建数据库（默认）:
  docker-compose -f docker-compose.yml -f docker-compose.self-db.yml up -d

外部数据库:
  docker-compose -f docker-compose.yml -f docker-compose.external-db.yml up -d

自建 + 低配 VPS:
  docker-compose -f docker-compose.yml -f docker-compose.self-db.yml -f docker-compose.lowmem.yml up -d

外部 + 低配 VPS:
  docker-compose -f docker-compose.yml -f docker-compose.external-db.yml -f docker-compose.lowmem.yml up -d

开发环境（自建数据库 + 热重载）:
  docker-compose -f docker-compose.yml -f docker-compose.self-db.yml -f docker-compose.dev.yml up
```

### 2.3 基础文件 `docker-compose.yml` 详解

只包含 `web` 服务，不含数据库：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `container_name` | `audiobook_web` | 容器名 |
| `build.dockerfile` | `docker/Dockerfile.web` | 多阶段构建 |
| `restart` | `unless-stopped` | 自动重启 |
| `dns` | `8.8.8.8`, `1.1.1.1` | 防 VPS DNS 解析失败 |
| `ports` | `59386:59386` | Web 端口 |
| `cpus` | `0.80` | CPU 限制 80%（防触发滥用检测） |
| `volumes` | `output_data:/data/output` | 输出持久化 |
| | `./data/music:/data/music` | BGM 音乐库（宿主机持久） |
| | `./data/deepfilter:/opt/deepfilter` | DeepFilter 二进制（宿主机持久） |
| | `./pipeline:/app/pipeline:ro` | Pipeline 源码只读挂载 |
| `logging` | `json-file`, 10m×3 | 日志大小限制 |

### 2.4 自建数据库覆盖 `docker-compose.self-db.yml`

添加 `postgres` 服务：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `container_name` | `audiobook_postgres` | 容器名 |
| `image` | `postgres:16-alpine` | PostgreSQL 16 Alpine 版 |
| `POSTGRES_DB` | `audiobook` | 数据库名 |
| `POSTGRES_USER` | `audiobook_app` | 数据库用户 |
| `POSTGRES_PASSWORD` | `${POSTGRES_PASSWORD}` | 从 .env 读取 |
| `volumes` | `postgres_data:/var/lib/postgresql/data` | 数据持久化 |
| | `./docker/init-db.sql:...:ro` | 初始化脚本（只读挂载） |
| | `./docker/postgresql-lowmem.conf:...:ro` | 低内存配置 |
| `command` | `postgres -c config_file=...` | 使用自定义配置 |
| `ports` | `5432:5432` | 暴露数据库端口 |
| `cpus` | `0.15` | CPU 限制 15% |
| `healthcheck` | `pg_isready -U audiobook_app -d audiobook` | 健康检查 |
| `web.depends_on` | `postgres: condition: service_healthy` | 等待数据库健康后启动 |
| `web.DATABASE_URL` | `postgresql://audiobook_app:${POSTGRES_PASSWORD}@postgres:5432/audiobook` | 覆盖为内部地址 |

### 2.5 外部数据库覆盖 `docker-compose.external-db.yml`

不启动 postgres 容器，直接连接外部数据库：

- `web.DATABASE_URL` = `${EXTERNAL_DATABASE_URL}`（从 .env 读取）
- `web.depends_on` = `[]`（不依赖任何服务）
- **前提**: 外部数据库已执行过 `docker/init-db.sql` 初始化表结构

### 2.6 低配覆盖 `docker-compose.lowmem.yml`

| 服务 | 限制 | 说明 |
|------|------|------|
| `postgres` | CPU 0.15, 内存 300M | 进一步压缩 PG 内存 |
| `postgres` | `shared_buffers=24MB, work_mem=1MB, max_connections=15` | 极限内存参数 |
| `web` | CPU 0.70, 内存 1400M（保留 400M） | Web 内存上限 |
| `web` | `VIDEO_RESOLUTION=720p` | 降低视频分辨率减少 CPU/内存 |
| `web` | 日志 5m×2 | 压缩日志 |
| `web.command` | `uvicorn ... --workers 1 --no-access-log` | 单进程 + 无访问日志 |

### 2.7 开发覆盖 `docker-compose.dev.yml`

- `web.command` = `uvicorn ... --reload --reload-dir /app/backend`（热重载）
- `web.volumes` = `./backend:/app/backend`（源码挂载，改代码即时生效）
- `web.environment` = `DEV_MODE=true`
- `postgres.ports` = `5432:5432`（暴露端口方便调试）

## 3. 环境变量配置（.env）

### 3.1 创建 .env

```bash
cp .env.example .env
nano .env
```

### 3.2 完整配置项

```bash
# ═══ 数据库模式 ═══
# self     = 使用 Docker 内置 PostgreSQL（默认，零配置）
# external = 连接外部已有的 PostgreSQL 实例
DB_MODE=self

# ═══ 自建数据库模式配置（DB_MODE=self 时使用）═══
# Docker 内置 PostgreSQL 的密码（首次启动时创建）
POSTGRES_PASSWORD=your_strong_password_here

# ═══ 外部数据库模式配置（DB_MODE=external 时使用）═══
# 填写外部 PostgreSQL 的完整连接串
# 注意：外部数据库需要已执行过 docker/init-db.sql 初始化表结构
# EXTERNAL_DATABASE_URL=postgresql://user:pass@external-host:5432/audiobook

# ═══ Web 服务 ═══
# 用于加密 OAuth Secret 等敏感数据的密钥（建议 64 位随机字符串）
# 生成方法：openssl rand -hex 32
SECRET_KEY=generate_a_random_64_char_string

# Web 界面访问地址（用于 OAuth 回调）
BASE_URL=http://localhost:59386

# Web 界面登录密码（默认 inriynisse）
# APP_PASSWORD=inriynisse
```

### 3.3 配置说明

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `DB_MODE` | 是 | `self` | `self` 或 `external` |
| `POSTGRES_PASSWORD` | `self` 模式必填 | `changeme_strong_password` | 内置 PG 密码 |
| `EXTERNAL_DATABASE_URL` | `external` 模式必填 | — | 外部 PG 连接串 |
| `SECRET_KEY` | 是 | `dev_secret_key_change_in_production` | 加密密钥 |
| `BASE_URL` | 是 | `http://localhost:59386` | OAuth 回调地址 |
| `APP_PASSWORD` | 否 | `inriynisse` | Web 登录密码 |

> **生产环境务必修改**: `POSTGRES_PASSWORD`、`SECRET_KEY`、`APP_PASSWORD`

## 4. 全新部署教程

### 步骤 1: 克隆项目

```bash
git clone <repo-url> yt_aduio_book_one_to_all
cd yt_aduio_book_one_to_all
```

### 步骤 2: 创建 .env 配置文件

```bash
cp .env.example .env
nano .env
```

修改以下关键项：
```bash
DB_MODE=self
POSTGRES_PASSWORD=你的强密码
SECRET_KEY=$(openssl rand -hex 32)
BASE_URL=http://你的服务器IP:59386
```

### 步骤 3: 使用部署脚本一键部署（推荐）

```bash
# 标准部署（自建数据库）
bash scripts/deploy.sh

# 强制重建镜像
bash scripts/deploy.sh --rebuild
```

部署脚本会自动完成：
1. 读取 `.env` 中的 `DB_MODE` 选择对应的 compose 覆盖文件
2. 构建 Docker 镜像
3. 启动服务（自建模式下 PostgreSQL 先启动并通过健康检查，然后启动 Web）
4. 等待 Web 服务就绪（最多 30 秒轮询 `http://localhost:59386/`）
5. 显示服务状态和访问地址

### 步骤 3 替代: 手动 Docker Compose 部署

```bash
# 自建数据库模式
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml up -d --build

# 外部数据库模式
docker-compose -f docker-compose.yml -f docker-compose.external-db.yml up -d --build
```

### 步骤 4: 验证服务启动

```bash
# 查看容器状态
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml ps

# 预期输出:
# NAME                 STATUS
# audiobook_postgres   Up (healthy)
# audiobook_web        Up
```

```bash
# 测试 Web 服务
curl http://localhost:59386/

# 查看 Web 日志
docker logs audiobook_web --tail 30

# 查看 PostgreSQL 日志
docker logs audiobook_postgres --tail 30
```

### 步骤 5: 验证数据库表结构

```bash
# 验证所有表已创建（应有 13 张表）
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "\dt public.*"

# 验证 books 表结构（应有 book_status 列）
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "\d books"

# 验证 audiobook_chapters 表结构（应有 worker_id/claimed_at/error_message 列）
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "\d audiobook_chapters"
```

### 步骤 6: 迁移数据（如需从旧项目导入）

详见 [第 8 节: 数据迁移方法](#8-数据迁移方法)。

### 步骤 7: 配置全局设置

打开浏览器访问 `http://你的服务器IP:59386`，使用密码登录（默认 `inriynisse`），进入 **配置管理** 页面配置以下项：

| 配置项 | 说明 |
|--------|------|
| `POSTGRES_DSN` | PostgreSQL 连接串（Pipeline 使用） |
| `TG_BOT_TOKEN` | Telegram Bot Token（多个用英文逗号分隔，支持多 Bot 轮换） |
| `TG_CHAT_ID` | Telegram Chat ID（音频缓存所在聊天/频道） |
| `MODELSCOPE_TOKEN` | ModelScope API Token（AI 封面生成） |
| `SENSENOVA_API_KEY` | Sensenova/DeepSeek API 密钥（Podcast 文案和封面） |
| `REMOTE_DATABASE_URL` | 远程 audiobook_pipeline 数据库 DSN（用于「从远程同步章节」功能） |

> 这些配置存储在 `global_settings` 表中，首次启动时由 `seed_global_settings()` 自动种子化默认值。

## 5. 低配 VPS (1C2G) 部署

适用于 1 核 CPU、2GB 内存的 VPS（如 CloudCone、RackNerd 等）。

### 一键部署

```bash
bash scripts/lowmem-deploy.sh
```

此脚本会自动完成：
1. 检查并创建 2GB Swap 文件（`scripts/setup-swap.sh`）
2. 调用 `scripts/deploy.sh --lowmem`（自动叠加 `docker-compose.lowmem.yml`）

### 手动部署

```bash
# 1. 创建 Swap（防止 OOM）
sudo bash scripts/setup-swap.sh

# 2. 启动低配模式
# 自建数据库 + 低配
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml -f docker-compose.lowmem.yml up -d --build

# 外部数据库 + 低配
docker-compose -f docker-compose.yml -f docker-compose.external-db.yml -f docker-compose.lowmem.yml up -d --build
```

### 低配资源分配

```
总内存 2GB
├── 系统 + Docker 守护进程  ~300MB
├── PostgreSQL             ~300MB (shared_buffers=24MB, max_connections=15)
├── Web (uvicorn 1 worker)  ~800MB (MALLOC_ARENA_MAX=2)
├── Swap 溢出保护           2GB
└── 余量                    ~600MB
```

| 限制项 | PostgreSQL | Web |
|--------|-----------|-----|
| CPU | 0.15 核 (15%) | 0.70 核 (70%) |
| 内存 | 300M | 1400M (保留 400M) |
| 日志 | — | 5m × 2 文件 |
| 视频分辨率 | — | 720p |
| Worker 数 | — | 1 |

## 6. 从旧版本升级

### 步骤 1: 停止服务

```bash
# 确定当前使用的 compose 文件组合
# 自建数据库模式:
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml down

# 外部数据库模式:
docker-compose -f docker-compose.yml -f docker-compose.external-db.yml down
```

### 步骤 2: 备份数据库（关键！）

```bash
# 完整备份数据库
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook > backup_$(date +%Y%m%d).sql

# 单独备份 books 和 audiobook_chapters 表
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook \
  -t books -t audiobook_chapters > backup_tables_$(date +%Y%m%d).sql
```

### 步骤 3: 拉取新代码

```bash
git pull origin main
```

### 步骤 4: 更新 .env（如有新增配置项）

```bash
# 对比 .env.example 和 .env，补充新增的配置项
diff .env.example .env
```

### 步骤 5: 使用服务器部署脚本（推荐）

```bash
# 在服务器上执行（自动 git pull + 智能构建 + 重启 + 健康检查）
bash scripts/git-server-deploy.sh
```

此脚本会自动完成：
1. `git pull` 拉取最新代码
2. 检查 `.env` 是否存在
3. 预下载 DeepFilter 二进制到 `./data/deepfilter/`（持久化，后续重建镜像不重复下载）
4. 预下载 BGM 音乐库到 `./data/music/`（持久化）
5. 智能判断是否需要重建镜像（对比 requirements.txt / Dockerfile / 源码 hash）
6. 重启服务
7. 健康检查（轮询 `http://localhost:59386/`）

### 步骤 5 替代: 手动升级

```bash
# 重建镜像并启动（自建数据库模式）
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml up -d --build

# 如果使用了低配模式
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml -f docker-compose.lowmem.yml up -d --build
```

> **init-db.sql 的 `ALTER TABLE ADD COLUMN IF NOT EXISTS` 会自动补充新列**，无需手动执行 SQL。如果数据库已存在（非首次启动），需要手动执行增量 SQL：

```bash
docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook <<'EOF'
-- books 表补充列
ALTER TABLE public.books ADD COLUMN IF NOT EXISTS book_status varchar(50) DEFAULT 'pending';
CREATE INDEX IF NOT EXISTS idx_books_category    ON public.books(category);
CREATE INDEX IF NOT EXISTS idx_books_book_status ON public.books(book_status);

-- audiobook_chapters 表补充列
ALTER TABLE public.audiobook_chapters ADD COLUMN IF NOT EXISTS telegram_bot_id integer;
ALTER TABLE public.audiobook_chapters ADD COLUMN IF NOT EXISTS telegram_bot_user_id bigint;
ALTER TABLE public.audiobook_chapters ADD COLUMN IF NOT EXISTS worker_id varchar(100);
ALTER TABLE public.audiobook_chapters ADD COLUMN IF NOT EXISTS claimed_at timestamptz;
ALTER TABLE public.audiobook_chapters ADD COLUMN IF NOT EXISTS error_message text;
CREATE INDEX IF NOT EXISTS idx_chapters_book_status ON public.audiobook_chapters(book_id, upload_status);
EOF
```

### 步骤 6: 验证升级

```bash
# 检查表结构
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "\d books"
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "\d audiobook_chapters"

# 检查数据统计
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "
  SELECT book_status, COUNT(*) FROM books GROUP BY book_status;
  SELECT upload_status, COUNT(*) FROM audiobook_chapters GROUP BY upload_status;
"

# 测试 Web 服务
curl http://localhost:59386/
```

## 7. 开发环境

### Windows

```bash
# 使用批处理脚本
scripts\dev.bat

# 或 PowerShell
scripts\dev.ps1
```

### Linux / macOS

```bash
# 手动启动（自建数据库 + 热重载）
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml -f docker-compose.dev.yml up
```

### 开发环境特性

- **热重载**: 修改 `backend/` 目录下的 Python 文件，容器内即时生效，无需重建镜像
- **源码挂载**: `./backend:/app/backend` — 本地修改直接映射到容器
- **Pipeline 只读挂载**: `./pipeline:/app/pipeline:ro`
- **数据库端口暴露**: `5432:5432` — 可用本地工具连接调试
- **DEV_MODE**: `DEV_MODE=true` 环境变量

> 开发模式下不加 `-d` 参数，日志直接输出到终端，`Ctrl+C` 停止。

## 8. 数据迁移方法

### 方法 A: 快速 SQL 管道迁移（推荐，速度快 10-50 倍）

```bash
cd migrate-tg-chapters

# 设置环境变量
export SOURCE_DATABASE_URL="postgresql://audiobook_app:inriynisse1991@85.121.48.55:5432/audiobook"
export DATABASE_URL="postgresql://audiobook_app:inriynisse1991@127.0.0.1:5432/audiobook"

# 迁移两张表（books + audiobook_chapters）
bash fast_migrate.sh --all

# 仅迁移 books
bash fast_migrate.sh --books

# 仅迁移 chapters（全部）
bash fast_migrate.sh --chapters

# 仅迁移整本完整的书（所有章节已上传到 TG）
bash fast_migrate.sh --chapters --only-complete-books

# 试运行（只统计，不写入）
bash fast_migrate.sh --all --dry-run

# 后台运行（断开 SSH 也不中断）
bash fast_migrate.sh --bg --all
```

> **原理**: 在单个 `postgres:16-alpine` 容器内，用 psql 管道直传 `COPY ... TO STDOUT | \copy ... FROM STDIN`，避免逐行 INSERT 的网络开销。

### 方法 B: Python 迁移脚本（支持灵活过滤）

```bash
cd migrate-tg-chapters

# 迁移 books 表
python migrate_books.py \
    --source-dsn "postgresql://audiobook_app:inriynisse1991@85.121.48.55:5432/audiobook" \
    --target-dsn "postgresql://audiobook_app:你的密码@127.0.0.1:5432/audiobook"

# 迁移 audiobook_chapters 表
python migrate_tg_chapters.py \
    --source-dsn "postgresql://audiobook_app:inriynisse1991@85.121.48.55:5432/audiobook" \
    --target-dsn "postgresql://audiobook_app:你的密码@127.0.0.1:5432/audiobook"

# 仅迁移已上传到 TG 的章节
python migrate_tg_chapters.py --only-uploaded

# 仅迁移整本全部上传到 TG 的书（推荐，确保数据完整）
python migrate_tg_chapters.py --only-complete-books

# 试运行预览
python migrate_books.py --dry-run
python migrate_tg_chapters.py --dry-run
```

### 方法 C: Docker 内运行（无需安装 Python）

```bash
cd migrate-tg-chapters

docker run --rm --add-host=host.docker.internal:host-gateway \
    -v "$(pwd)/migrate_books.py:/app/migrate_books.py" \
    -w /app \
    python:3.12-slim bash -c "
        pip install 'psycopg[binary]' -q
        python migrate_books.py \
            --source-dsn 'postgresql://audiobook_app:inriynisse1991@85.121.48.55:5432/audiobook' \
            --target-dsn 'postgresql://audiobook_app:你的密码@host.docker.internal:5432/audiobook'
    "
```

### 方法 D: 同一 docker-compose 内迁移（从旧库到新库）

```bash
# 使用 migrate_data.sh 脚本（迁移 6 张核心表）
# 适用于从宿主机旧 PostgreSQL 迁移到 Docker 内 PostgreSQL
OLD_PG_HOST=127.0.0.1 OLD_PG_PORT=5432 OLD_PG_USER=postgres \
OLD_PG_DB=audiobook OLD_PG_PASSWORD=旧库密码 \
    bash scripts/migrate_data.sh
```

### 方法 E: pg_dump / pg_restore（全量导入）

```bash
# 从源数据库导出 books + audiobook_chapters 的数据
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook \
  -t books -t audiobook_chapters --data-only > pipeline_data.sql

# 导入到目标数据库
docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook < pipeline_data.sql
```

### 迁移注意事项

> **`status` 列 vs `book_status` 列**：
> - `status` — 远程系统状态 + 项目 flag（防频道重复处理），迁移时保留原值
> - `book_status` — audiobook_pipeline 章节完成标记（pending/success），迁移时保留原值
>
> 迁移脚本已适配所有新增字段（`book_status`、`worker_id`、`claimed_at`、`error_message`、`telegram_bot_id`、`telegram_bot_user_id`）。

## 9. 书籍管理功能按钮

Web 面板「书籍管理」页面提供以下操作按钮：

### 9.1 回填分类

**按钮**: 🔄 回填分类
**API**: `POST /api/books/backfill-category`

从 `book_data` JSON 中提取分类回填到 `category` 顶层列。尝试多种字段名（`category` / `bookCategory` / `tingCategory` / `categoryId` 等），兼容顶层和 `bookInfo` 嵌套结构。只更新 `category` 为空的记录。

### 9.2 同步章节状态

**按钮**: ✅ 同步章节状态
**API**: `POST /api/books/sync-book-status`

扫描 `audiobook_chapters` 表，找出所有章节均已成功上传到 Telegram 的书籍（所有章节 `upload_status='uploaded'`），将其 `books.book_status` 标记为 `success`。不影响 `status` 列。

### 9.3 从远程同步章节

**按钮**: ☁️⬇️ 从远程同步章节
**API**: `POST /api/books/sync-chapters-from-remote`

从远程 audiobook_pipeline 数据库（`REMOTE_DATABASE_URL` 配置项指定的地址）读取 `upload_status='uploaded'` 的章节，将其 TG 缓存信息（`telegram_file_id`、`telegram_bot_id` 等）批量同步到本地 `audiobook_chapters` 表。

**适用场景**: 远程 Worker 完成音频处理并上传到 Telegram 后，将 TG 文件 ID、Bot 信息拉取到本地，供本地从 Telegram 下载已降噪音频。

**同步字段**: `telegram_file_id`、`telegram_message_id`、`telegram_bot_id`、`telegram_bot_user_id`、`upload_status`、`uploaded_at`、`error_message`

**DSN 优先级**: `global_settings` 表的 `REMOTE_DATABASE_URL` → 环境变量 → 内置默认值 `postgresql://audiobook_app:inriynisse1991@85.121.48.55:5432/audiobook`

> 远程库地址可在「配置管理」页面修改 `REMOTE_DATABASE_URL`。

## 14. 常见问题

### Q1: 容器名是什么？

- Web 容器: `audiobook_web`
- PostgreSQL 容器: `audiobook_postgres`
- 服务名（docker-compose 内部）: `web`、`postgres`

### Q2: Web 端口是多少？

端口 **59386**。docker-compose.yml 中映射为 `59386:59386`。

### Q3: 为什么 `docker compose up -d` 不起作用？

本项目使用多文件组合模式，必须指定所有需要的 compose 文件：

```bash
# 错误（只有 web，没有数据库）
docker-compose up -d

# 正确（自建数据库）
docker-compose -f docker-compose.yml -f docker-compose.self-db.yml up -d

# 或使用部署脚本（自动选择）
bash scripts/deploy.sh
```

### Q4: 迁移后 Web 面板分类为空？

**原因**: 迁移的数据中 `category` 列可能为空。

**解决**: 在 Web 面板点击「回填分类」按钮，或调用 API：

```bash
curl -X POST http://localhost:59386/api/books/backfill-category
```

### Q5: `book_status` 列全是 `pending`？

**正常现象**。`book_status` 是 audiobook_pipeline 的章节完成标记。在 Web 面板点击「同步章节状态」按钮，系统会自动扫描章节上传状态，将所有章节已上传的书标记为 `success`。

```bash
curl -X POST http://localhost:59386/api/books/sync-book-status
```

### Q6: 如何从远程 audiobook_pipeline 数据库同步章节信息到本地？

在 Web 面板点击「从远程同步章节」按钮，或调用 API：

```bash
curl -X POST http://localhost:59386/api/books/sync-chapters-from-remote
```

该功能会连接远程 audiobook_pipeline 数据库，读取已上传章节的 `telegram_file_id`、`telegram_bot_id` 等 TG 缓存信息，批量更新到本地 `audiobook_chapters` 表。远程库地址在「配置管理」页面的 `REMOTE_DATABASE_URL` 配置项中修改。

### Q7: `status` 列的 flag 机制是否受影响？

**不受影响**。`status` 列继续承担远程状态 + 项目标记 flag 的双重职责：
- `finalize_successful_book_for_project` 继续用 `build_supabase_text_update` 往 `status` 追加 flag
- `_update_book_status_in_database` 继续更新 `status` 列
- 防频道重复处理/上传的逻辑完全保留

`book_status` 仅是 audiobook_pipeline 自身的章节完成标记，与 `status` 完全独立。

### Q8: `fast_migrate.sh` 报错？

**常见原因**: Docker 容器无法访问宿主机数据库。

**解决**:
```bash
# 确保 DATABASE_URL 使用 host.docker.internal 而非 127.0.0.1
export DATABASE_URL="postgresql://audiobook_app:你的密码@host.docker.internal:5432/audiobook"

# 或者如果数据库在 Docker 网络中
export DATABASE_URL="postgresql://audiobook_app:你的密码@audiobook_postgres:5432/audiobook"
```

### Q9: DeepFilter 或 BGM 音乐下载失败？

**DeepFilter**: 部署脚本 `git-server-deploy.sh` 会预下载到 `./data/deepfilter/`。容器启动时 `entrypoint.sh` 也会自动检查并下载。如果网络不稳定，可手动下载：

```bash
wget "https://github.com/Rikorose/DeepFilterNet/releases/download/v0.5.6/deep-filter-0.5.6-x86_64-unknown-linux-musl" \
    -O data/deepfilter/deep-filter-0.5.6-x86_64-unknown-linux-musl
chmod +x data/deepfilter/deep-filter-0.5.6-x86_64-unknown-linux-musl
```

**BGM 音乐**: 部署脚本预下载到 `./data/music/`。容器启动时 `entrypoint.sh` 也会自动检查并下载。

### Q10: 外部数据库模式如何初始化表结构？

外部数据库需要手动执行 `init-db.sql`：

```bash
psql "postgresql://user:pass@external-host:5432/audiobook" -f docker/init-db.sql
```

### Q11: 如何修改 PostgreSQL 密码？

修改 `.env` 中的 `POSTGRES_PASSWORD`，然后删除数据卷重建：

```bash
# ⚠️ 警告：这会删除所有数据！先备份！
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook > backup.sql

docker-compose -f docker-compose.yml -f docker-compose.self-db.yml down
docker volume rm yt_aduio_book_one_to_all_postgres_data

# 修改 .env 中的 POSTGRES_PASSWORD
nano .env

docker-compose -f docker-compose.yml -f docker-compose.self-db.yml up -d --build

# 恢复数据
docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook < backup.sql
```
